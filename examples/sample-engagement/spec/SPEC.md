## Acceptance Criteria

1. The checkout is a single page presenting, in order: cart summary, delivery address, payment, place order. [claim-005 · sources/kickoff-call.md:L23]
2. The payment step offers Apple Pay, Google Pay, and the existing manual card form. [claim-006 · sources/product-notes.md:L12]
3. On a mobile viewport, the wallet options render as the primary, most-prominent payment choice above the card form. [claim-007 · sources/product-notes.md:L24]
4. Every placed order is associated with a signed-in Forkful account; signup is collected inline in the address step with no separate signup screen. [conflict-001, claim-009 · decisions/2026-06-13-resolution-conflict-001.md]
5. The cart summary displays a delivery ETA that recalculates whenever the delivery address changes. [claim-015 · sources/product-notes.md:L20]
6. On successful order placement, the user lands on a confirmation screen showing the order number, estimated delivery time, and a "we're on it" message. [claim-011 · sources/kickoff-call.md:L37]
7. Each completed order accrues loyalty points at 1 point per dollar, surfaced to the user after completion. [claim-016 · sources/product-notes.md:L27-L28]
8. A declined payment surfaces an explicit error message rather than silently re-presenting the form. [claim-010 · sources/kickoff-call.md:L35]
9. The checkout reduces the add-to-cart → order-placed drop-off; success is measured as drop-off below 20% (from ~31%). [claim-002 · sources/product-notes.md:L6]

## Non-Goals

- Cart recovery / abandoned-cart email is out of scope for this release. [claim-013 · sources/product-notes.md:L32]
- Broad in-session "don't lose the cart" work beyond the checkout itself is out of scope ("let's not boil the ocean"). [claim-014 · sources/kickoff-call.md:L43]
- Literal anonymous guest orders (an order with no account) are not built; the account model requires an account on every order. [conflict-001 · decisions/2026-06-13-resolution-conflict-001.md]

## Edge Cases

- **Declined / failed payment:** must show an actionable error state, not a silent re-render of the form (current behavior causes retries and rage-quits). No design frame exists yet for this state. [claim-010, gap-002 · sources/kickoff-call.md:L35]
- **Address change after ETA shown:** the delivery ETA on the cart summary must recompute on address change. [claim-015 · sources/product-notes.md:L20]
- **Saved cards (UNRESOLVED):** behavior and interaction with the account/inline-signup model is parked and must be settled before the payment step is implementable. [gap-001 · sources/kickoff-call.md:L31]
- **Promo code (UNDERSPECIFIED):** entry point, validation, stacking rules, and effect on the displayed total are not yet specified. [gap-004 · sources/product-notes.md:L35]
- **Mid-checkout cart persistence (UNDEFINED):** whether the cart survives a refresh or navigation away during the single-page flow is undefined. [gap-005 · sources/kickoff-call.md:L43]

## Data / API Contracts

### Order

- `order.accountId: string` — every order is tied to a signed-in account; this field is always present. [conflict-001, claim-009 · decisions/2026-06-13-resolution-conflict-001.md]
- `order.orderNumber: string` — surfaced on the confirmation screen. [claim-011 · sources/kickoff-call.md:L37]
- `order.estimatedDeliveryTime: string` — surfaced on the confirmation screen and on the cart summary as an ETA. [claim-015, claim-011 · sources/product-notes.md:L20]
- `order.loyaltyPointsEarned: number` — 1 point per dollar of order total, computed on completion. [claim-016 · sources/product-notes.md:L27-L28]

### Payment

- `payment.method: "apple_pay" | "google_pay" | "card"` — the supported payment methods. [claim-006 · sources/product-notes.md:L12]
- `payment.walletPrimaryOnMobile: boolean` — wallets are the primary option on mobile; defaults true on mobile viewports. [claim-007 · sources/product-notes.md:L24]
- `payment.declineMessage: string` — populated on a declined payment so the UI can show an explicit error state. [claim-010 · sources/kickoff-call.md:L35]

### Cart

- `cart.deliveryEta: string` — shown on the cart summary, recalculated on address change. [claim-015 · sources/product-notes.md:L20]
- `cart.promoCode: string | null` — promo / discount codes must continue to function; full handling is still unspecified. [claim-012, gap-004 · sources/product-notes.md:L35]
