# Sample Engagement — Forkful Checkout Revamp

A small, self-consistent **fictional** product engagement used as the dogfood corpus and test
fixture for the Forward-Momentum pipeline. It mirrors the engagement-repo layout from
[`SPEC.md`](../../SPEC.md) §"Project Structure".

## The engagement

**Forkful** is a (fictional) food-delivery app. Checkout conversion is bleeding — ~31% drop-off
between "add to cart" and "order placed" — and the team is running a Q3 revamp to rebuild the
checkout flow (guest checkout, wallets, fewer steps) to push drop-off under 20%.

## Layout

```
sources/        The messy inputs YOU read into the pipeline (kickoff call, notes, design refs).
analysis/       Generated by /fm-extract + /fm-gaps  → claims.json, gap-report.md  (empty: .gitkeep)
decisions/      Generated: ADR-style decision log, gate records, waivers          (empty: .gitkeep)
prd/            Generated by /fm-prd: human narrative view                          (empty: .gitkeep)
spec/           Generated by /fm-prd: machine view (testable AC, contracts)         (empty: .gitkeep)
tasks/          Generated by /fm-tasks: design tasks + dispatch state               (empty: .gitkeep)
.flow/          Flow state machine (state.json) — initialized by the flow controller (empty: .gitkeep)
.claude/        Engagement-local skill-pack copy                                    (empty: .gitkeep)
ANSWER-KEY.md   Test oracle: every deliberately planted issue, with where it hides.
```

## How to use it

- **Pipeline / demo:** point the extension (or the standalone `claude` skill-pack) at this directory
  as the engagement repo. The generated artifacts land in `analysis/`, `decisions/`, `prd/`, etc.
- **Golden test (T5):** `sources/` contains **deliberately planted gaps and one conflict**. The
  gap-analysis golden test asserts `/fm-gaps` surfaces every issue enumerated in
  [`ANSWER-KEY.md`](./ANSWER-KEY.md). That answer key is the oracle — keep it in sync with `sources/`.
